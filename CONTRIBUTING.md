# Contributing

Thank you for your interest in contributing.

We keep the contributor circle for this project intentionally small. While the project is open source, we only accept outside pull requests selectively.

We prefer contributions from people who are actual users of the software and have direct experience with the problem it solves.

## Pull requests

We generally only consider non-trivial pull requests from outside contributors, such as:

* Bug fixes for issues encountered through real usage
* Meaningful documentation improvements based on actual use
* Compatibility fixes
* Tests covering real behavior or regressions
* Well-scoped improvements discussed with maintainers beforehand

We usually do not accept:

* Typo-only pull requests
* Formatting-only changes
* Cosmetic changes
* Drive-by refactors
* Pull requests from people who do not use the software
* Contributions opened mainly to build contribution history

Small issues, such as typos, may be reported as issues instead. Maintainers may choose to fix them internally.

## Before contributing

For anything other than a clear bug fix, please open an issue first and explain how you use the software, what problem you encountered, and what change you propose.

Maintainers may close pull requests without merging them, even if the change is technically correct, if they do not fit the project’s scope, maintenance model, or contribution policy.

## Licensing

By submitting a contribution, you agree that it is provided under the same license as this repository and confirm that you have the right to submit it.

We may request additional sign-off or licensing confirmation before reviewing or merging a contribution.
